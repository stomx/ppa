# AX 조직 내 성장 계획

---

## 1. 성장 방향 요약

AX 조직의 현재 개발 단계에 맞춰,  
**AX Studio 및 AX Agent 개발 과정에서 이미 수행해 온 프론트엔드 역할을 중심 축으로 유지**함.

이를 기반으로,  
프론트엔드·백엔드 개발 경험과 AI Agent 구현 경험을 연결하여  
**AX 조직의 개발 경험(DX)을 정립하고, 에이전트 제품 전반의 품질과 일관성을 높이는 데 기여**하고자 함.

기술적 안정성과 개발 효율을 체계적으로 확보하는 것은  
**개발 속도 향상, 변경 비용 감소, 그리고 비즈니스 의사결정의 민첩성 확보로 이어진다고 판단**함.

---

## 2. 단기 계획 (0~3개월)

### 2-1. AX Studio 프론트엔드 개발 책임 지속

AX Studio 개발 과정에서 맡아온 프론트엔드 역할을 지속하며,  
**제품 개발 과정에서 프론트엔드 구현의 기준을 안정적으로 유지**함.

- AX Studio 전반의 화면 구조, 페이지 구성, 공통 컴포넌트 사용 방식에 대한 기준 유지
- 누적된 구현 히스토리와 맥락을 바탕으로 한 프론트엔드 개발 대응

---

### 2-2. 프론트엔드 개발 기준 및 DX 구체화

현재 명확히 정리되어 있지 않은 프론트엔드 개발 방식을 구체화하여,  
**AX 조직에 적합한 개발 기준과 작업 흐름을 정립**함.

- 프론트엔드 책임 범위(화면, 상태, 사용자 인터랙션) 명확화
- 스키마 선행 정의, Mock/Fake API 활용 등을 포함한 Frontend-Driven 개발 방식 정리
- 프론트엔드–백엔드 병렬 개발이 가능하도록 작업 순서와 산출물 기준 정의
- 논의를 통해 합의된 내용을 문서화하여 AX의 DX로 정착

---

### 2-3. 프론트엔드–백엔드 간 인터페이스 기준 정리

각 직군이 자신의 전문 영역에 집중할 수 있도록,  
**프론트엔드–백엔드 간 인터페이스 기준을 명확히 정리**함.

- 프론트엔드는 사용자 인터랙션, 화면 표현, 상태 관리에 집중
- 백엔드는 비즈니스 로직, 데이터 무결성, 성능, 인프라에 집중
- API 스펙, 요청·응답 구조, 에러 처리 방식 등에 대한 기준 정리
- 스키마 정의 및 Migration Guide를 통해 협업 시 혼선 최소화

---

## 3. 중기 계획 (3~12개월)

### 3-1. AX Agent 및 Agent Builder 개발 맥락 기여

AX가 AI Agent 및 Agent Builder를 핵심 제품으로 발전시키는 과정에서,  
**기존에 보유한 Agentic AI 구현 경험을 실제 개발 논의와 설계 판단에 연결**함.

- STT, LLM, TTS, VAD 기반 AI 웹서비스 및 Agent 구현 경험 공유
- Agent 실행 흐름, 상태 관리, 사용자 인터랙션 설계에 대한 실무 관점 의견 제공
- Agent Builder 사용 흐름에 대한 프론트엔드·UX 관점 피드백 제공

---

### 3-2. 개발 효율 및 협업 기여

개발 과정에서 발생하는 반복 작업과 불필요한 커뮤니케이션 비용을 줄이기 위해  
**개발 효율과 협업 품질 향상에 기여**함.

- WebApp Generator 등 생산성 도구 활용 경험 공유
- SI 및 외부 프로젝트 대응 과정에서 재사용 가능한 구조와 작업 방식 정리
- 개발 맥락, 결정 배경, 작업 기준을 문서로 남겨 협업 시 참고 가능하도록 유지
- 요청이 있는 경우 코드 리뷰 및 구현 방향에 대한 기술적 의견 제공

---

## 4. 장기 관점

### 4-1. 풀스택 역량을 활용한 에이전트 제품 품질 기여

프론트엔드뿐 아니라 백엔드 개발 경험을 보유한 풀스택 개발자로서,  
**에이전트 개발 과정에서 전체 흐름을 고려한 품질 개선에 기여**함.

- 프론트엔드–백엔드 양측을 이해한 상태에서 설계·구현 간 괴리 최소화
- Agent 실행 로직, 데이터 흐름, UI 인터랙션 간 정합성 관점의 기술적 피드백 제공
- 기능 단위가 아닌, 사용자 경험과 시스템 동작을 함께 고려한 품질 판단 기여

---

### 4-2. 개발 자산 및 경험 축적

개인의 경험에 의존하지 않고,  
**AX 조직 내에 개발 기준과 맥락이 누적되는 구조를 유지**함.

- 프론트엔드·백엔드·Agent 개발 경험, 설계 판단, 구현 패턴의 문서화
- 신규 인원이 합류하더라도 빠르게 맥락을 이해할 수 있는 개발 자산 유지
- AX 조직의 에이전트 개발 역량이 조직 단위로 축적되도록 기여
